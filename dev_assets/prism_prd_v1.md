## §01 배경과 문제

D2C 미디어커머스에서 마케팅 콘텐츠는 곧 매출이다. 하지만 콘텐츠를 만드는 과정은 세 가지 분절된 활동으로 흩어져 있고, 각 활동의 결과물이 서로 연결되지 않는다. Prism은 이 세 가지를 하나의 컨텍스트로 묶는다.

### Problem 01 · 분산된 리서치
**경쟁사 광고 리서치가 손작업으로 흩어진다**

마케터는 매주 Meta Ad Library, TikTok Creative Center, YouTube Ads, Instagram을 따로따로 모니터링한다. 발견한 좋은 광고는 스크린샷·노션·슬랙으로 흩어지고, 한 달 후엔 어디 있는지 못 찾는다. 갤러리화·구조화된 컬렉션이 없다.

### Problem 02 · 기획 단계의 병목
**콘텐츠 기획 → 검수의 사이클이 너무 길다**

사내에는 콘텐츠 가이드 문서가 존재하고 타율 높은 콘텐츠 패턴도 알려져 있지만, 기획자가 작성한 초안이 그 가이드와 어긋나는 경우가 많다. 팀 리더가 매번 검수·피드백·재기획 사이클을 돌리며 병목이 발생한다. 대본·이미지 배치·소구점 결정에 들어가는 리소스가 상당하다.

### Problem 03 · 데이터 단절
**콘텐츠 결정과 성과 데이터가 분리되어 있다**

광고 성과 지표(ROAS, CTR, CVR), 유저 행동 데이터(Funnel, Retention), 콘텐츠 의사결정이 각각 다른 도구·다른 화면에 산다. 어떤 콘텐츠가 어떤 유저 행동을 만들었고, 그게 어떤 결과 지표로 이어졌는지 — 이 인과 사슬을 자연어로 묻고 답할 수 있는 도구가 없다.

## §02 타겟 사용자

마케팅팀 구성원에서 팀 리더까지 — 같은 화면을 본다. 권한 분리는 없고, 역할에 따라 사용 빈도가 다른 영역이 다를 뿐.

| 역할 | 주된 활동 | 자주 쓰는 모듈 |
|---|---|---|
| 콘텐츠 마케터 | 광고 콘텐츠 기획 · 카피 작성 · 콘티 제작 | 갤러리 · 어시스턴트 |
| 퍼포먼스 마케터 | 광고 운영 · 성과 모니터링 · 채널 최적화 | 대시보드 · 갤러리 |
| 마케팅 팀 리더 | 전체 방향성 검토 · 의사결정 · 분기 리뷰 | 홈 · 대시보드 · 어시스턴트(검수) |

**설계 원칙:** 누가 쓰든 같은 화면. 검수 흐름의 경우 자연스럽게 기획자가 1차 어시스턴트 검수 → 팀 리더가 결과를 함께 보는 구조로 작동하지만, 별도 권한 분리는 두지 않는다.

## §03 제품 비전 · UVP

Prism은 세 가지 기능을 통합한 하나의 제품이다 — 콘텐츠 OS, 광고 콘텐츠 기획 가속기, 마케팅 인텔리전스 워크스페이스. 단순한 기능 묶음이 아니라, **AI가 모든 모듈을 관통하는 두뇌**로 작동한다는 점이 차별점.

### 01. Research — 경쟁사 갤러리
Meta · IG · YouTube · TikTok의 경쟁사 광고를 한 곳에서 갤러리뷰로. 브랜드·제품·국가·채널로 필터링, 광고 콘티·컨셉·주요 지표까지 구조화. AI가 선택된 광고들의 공통 패턴을 자동 분석.

### 02. Plan — AI 콘텐츠 검수
사내 콘텐츠 가이드와 타율 데이터가 LLM에 내장된 어시스턴트. 기획서 업로드 한 번이면 가이드 일치도 점수, 항목별 코멘트, 개선 카피 + 근거, 씬 구성, 타사 비교, 사내 타율 상위 비교까지 풀패키지 답변.

### 03. Analyze — 인과 분해 대시보드
Amplitude 연동 가정. 결과 지표(ROAS, 매출) ↔ 선행 지표(CTR, CVR, 노출) ↔ 유저 행동 데이터의 인과 분해를 4가지 시각화 방식으로. 자연어 챗봇으로 "왜 ROAS가 떨어졌어?" 묻기.

### 차별점

- **한 컨텍스트, 세 활동.** 갤러리에서 발견한 광고를 어시스턴트로 보내 검수 인풋으로 사용, 어시스턴트의 결과를 대시보드 시뮬레이션으로 연결, 대시보드의 인사이트를 다시 갤러리/어시스턴트로 — 모듈 간 컨텍스트가 자연스럽게 흐름.
- **AI가 옆에 항상 있다.** 좌측 하단 floating 버튼 → 클릭 시 사이드패널. 현재 보고 있는 모듈에 따라 추천 질문이 달라지고, 답변 안에서 다른 모듈로 직접 이동하는 액션 칩.
- **마케터가 직접 만든다.** 분석 → 검수 → 결정의 사이클을 IT/외주에 의존 없이 마케터 본인이 운영한다.

## §04 정보 구조

하나의 사이트, 4개 탭, 모든 페이지에 떠있는 AI 사이드패널 버튼, 그리고 모듈 사이를 잇는 컨텍스트 전달 다리들.

### 전체 레이아웃

```
┌──────────────────────────────────────────────────┐
│ Prism · 홈 갤러리 어시스턴트 [대시보드]   ● Connected │
├──────────────────────────────────────────────────┤
│                                                  │
│         메인 컨텐츠 영역 — 선택된 탭의 모듈           │
│                                                  │
│   ┌─┐                                            │
│   │✦│ ← 좌하단 floating AI 버튼 (모든 페이지)        │
│   └─┘    클릭 → 우측 사이드패널 슬라이드 인           │
└──────────────────────────────────────────────────┘
```

### 모듈 간 컨텍스트 다리

한 모듈에서 발견한 것을 다른 모듈로 한 번에 넘기는 4개의 다리. **시연의 핵심.**

| 다리 | 흐름 |
|---|---|
| `갤러리 → 어시스턴트` | 경쟁사 광고 5개 선택 → "이 컨셉으로 우리 광고 만들기" → 어시스턴트로 자동 컨텍스트 전달, 검수 화면에 인풋으로 채워짐 |
| `어시스턴트 → 대시보드` | 검수 완료 후 "예상 ROAS 시뮬레이션 보기" → 대시보드로 이동, 개선안 적용 시 예상 임팩트가 인사이트 카드로 표시 |
| `대시보드 → 어시스턴트` | AI 인사이트 카드의 "이 인사이트로 새 콘텐츠 기획" 칩 클릭 → 어시스턴트로 이동, 인사이트가 컨텍스트로 입력됨 |
| `대시보드 → 갤러리` | "비슷한 컨셉의 경쟁사 광고 보기" 칩 클릭 → 갤러리로 이동, 인사이트 키워드로 자동 필터링된 카드들 노출 |

## §05 비주얼 디자인

Amplitude 홈페이지 톤을 베이스로, 정교한 그라데이션 / 한국어 타이포 / 글래스 카드 / 마이크로 인터랙션으로 한 단계 위로. **모듈마다 약간 다른 무드** — 갤러리는 매거진, 어시스턴트는 작업실, 대시보드는 브리핑 룸, 홈은 다크 그라데이션.

### 컬러 토큰

| 토큰 | HEX | 용도 |
|---|---|---|
| `BG` | `#FAFAF7` | 페이퍼 베이스 배경 |
| `INK` | `#141412` | 메인 텍스트, 다크 카드 |
| `INDIGO` | `#5B4FE5` | 메인 액센트 (AI · 액션) |
| `PINK` | `#EC4899` | 서브 액센트 (하이라이트) |
| `GREEN` | `#10B981` | 성공 · 긍정 시그널 |
| `AMBER` | `#D97706` | 경고 · 주의 |

### 타이포그래피

| 역할 | 폰트 | 비고 |
|---|---|---|
| 한글 본문 · UI | `Pretendard` 400/500/600 | 모든 본문 |
| 영문 디스플레이 | `Fraunces` 300/400/500 | 이탤릭 적극 사용 |
| 코드 · 라벨 · 메타 | `JetBrains Mono` 500 | 섹션 라벨, 타임코드 등 |

### 컴포넌트 스타일

- **그라데이션 메쉬**: 단색 배경 대신 부드러운 다층 그라데이션 (보라 → 인디고 → 마젠타). 노이즈 그레인 텍스처 살짝.
- **플로팅 카드**: 흰색 배경, 라운드 16~24px, 부드러운 그림자, 호버 시 살짝 떠오르는 마이크로 인터랙션.
- **글래스 효과 (선택)**: AI 사이드패널은 backdrop-blur로 살짝 비치는 느낌.
- **모듈별 무드**: 갤러리 = 라이트 + 매거진톤 / 어시스턴트 = 라이트 + 작업실톤 / 대시보드 = 다크 액센트 + 브리핑톤 / 홈 = 다크 그라데이션.
- **아이콘**: Tabler Icons Outline (일관된 stroke).

### 반응형 원칙

데스크톱(1280px+)을 1차 타겟. 태블릿(768px+)에서는 사이드패널·필터를 collapsible로 전환. **모바일은 오늘 우선순위 아님** (Out of Scope §08 참조).

## §06 모듈 명세

4개 메인 모듈 + 1개 cross-cutting 컴포넌트(AI 사이드패널). 각 모듈의 화면 구성, 컴포넌트, 데모 데이터 구조, 인터랙션을 명세.

### 6.1 홈 대시보드

> `welcome · briefing · entry`

**목적** — 사용자 진입 시 오늘의 컨텍스트 한 페이지 제공. AI 브리핑 + 모듈 진입 카드 + 액션 카드.

**레이아웃**
- 상단: 인사 + AI 브리핑
- 중단: 4개 모듈 진입 카드 (각 모듈의 최근 활동 미리보기)
- 하단: "지금 해야 할 일" 액션 카드 + 주요 KPI 요약

**상단 영역 — 환영 메시지 + AI 브리핑 한 단락**
> "어제 8개 새 경쟁사 광고가 추가됐어요. ROAS는 3.8 (+12%). 검수 대기중인 콘텐츠 3건. 화장솜 카테고리 콘텐츠 피로도 신호 감지됨."

**중단 — 모듈 카드 4개**
- **갤러리** — "어제 새 광고 8건. 활발한 트렌드 — Before/After 컨셉"
- **어시스턴트** — "검수 대기 3건. 평균 가이드 일치도 71%"
- **대시보드** — "ROAS 3.8 · 일주일 +0.4. 신규 가입 코호트 D7 retention +12%p"
- **설정/가이드** — "이번주 가이드 v3.3 업데이트"

**하단 — 액션 카드**
- "3건 검수 대기 중 — 지금 처리하기" → 어시스턴트로
- "ROAS 하락 신호 — 원인 분석하기" → 대시보드로
- "새 경쟁사 광고 8건 — 갤러리에서 보기" → 갤러리로

**비주얼 톤** — Amplitude 메인 페이지와 가장 유사한 톤. 다크 그라데이션 배경 가능, 큰 한글 헤딩 + 흰색 floating 카드들. **시연 시작 임팩트가 가장 큰 곳.**

### 6.2 갤러리

> `pinterest masonry · ad research`

**목적** — 경쟁사의 IG · 유튜브 · 틱톡 · Meta 광고를 한 곳에서 갤러리뷰로 탐색 · 분류 · 분석.

**레퍼런스** — `foreplay.co` (스와이프 파일/Boards 구조) · Pinterest masonry 그리드 · Meta Ad Library (광고 메타데이터 표시)

**화면 구성**
- **좌측 사이드바**: 필터 (브랜드 / 제품 / 국가 / 채널 / 플랫폼)
- **상단 검색바 + 정렬**: 검색, 정렬 옵션 (최신/인기/타율 점수)
- **메인 그리드**: Masonry 레이아웃, 카드 가변 높이
- **카드 클릭 시**: 우측 디테일 패널 슬라이드 (모달 또는 사이드뷰)
- **하단 floating 액션바**: 카드 다중 선택 시 등장 — "5개 선택 · AI 패턴 분석" 버튼

**카드 정보**
썸네일 (호버 시 영상 자동 재생) · 브랜드 로고/이름 · 플랫폼 아이콘 · 카피 첫 1~2줄 · 컨셉 한 줄 · 콘티 요약 · 주요 지표 (예상 노출/광고비/CTR) · 운영 일수 · 타율 점수 · 카테고리 태그

**디테일 패널**
확대 썸네일 · 풀 카피 · 콘티 (씬별) · 컨셉 분석 · 소구점 추출 · 메트릭 풀 · "이 광고로 우리 콘텐츠 만들기" 버튼 (어시스턴트 컨텍스트 전달)

**AI 자동 분석**
다중 선택 → 패턴 분석 → 모달 노출. 공통 후크 / 공통 소구점 / 공통 CTA / 추천 적용 방향. 모달 하단에 "어시스턴트로 보내기" 버튼.

**데모 데이터**
가공 경쟁사 5~8개 (`GLOWTH`, `BREU`, `VITAVITA`, `MELLO`, `SKINEDIT` 등). 광고 30개 분량. JSON 스키마 아래 참조.
> ※ **광고 이미지·영상 샘플은 푸우가 직접 캡처하여 빌드 단계에 제공.**

### 6.3 어시스턴트

> `notion-AI tone · full review`

**목적** — 사내 콘텐츠 가이드 + 타율 데이터를 컨텍스트로 가진 AI 어시스턴트가 기획서를 검수하고, 카피·콘티·이미지·영상 가이드까지 풀패키지 답변.

**레퍼런스** — Notion AI 채팅 톤 (진입 화면) · Writer.com 브랜드 보이스 거버넌스 (검수 결과) · Claude Projects (컨텍스트 시스템)

#### 진입 상태 (State 1)

- 중앙에 큰 헤딩 — "무엇을 도와드릴까요?"
- **채팅바가 메인 인터페이스.** paperclip 아이콘으로 파일/이미지/URL 업로드 통합
- 채팅바 하단 시작 카드 5개: `기획서 검수` / `대본 작성` / `컨셉 제안` / `소구점 분석` / `타사 광고 비교`
- 노션 AI 톤 — 미니멀, 화이트 베이스

#### 업로드 후 상태 (State 2)

화면이 좌우 두 패널로 분리된 **워크스페이스 모드**로 자동 전환.

- **좌측 (sticky, 280px)** — 내가 올린 기획서 풀 본문
- **우측 (스크롤)** — AI 검수 결과 6개 영역

#### 좌측 패널 — 기획서 풀 본문

요약본이 아니라 **풀 본문**이 표시됨. 우측 검수 결과와 직접 비교 가능.

- 제목 / 컨셉 / 타겟 / 채널 / 소구점
- 콘티 (씬별 상세)
- **대본 전문** (시간 코드 포함)
- 이미지 배치 계획 (각 씬에 들어갈 비주얼)
- CTA 카피

페이지 스크롤 시 좌측은 sticky로 고정. 우측 검수 결과를 보면서 좌측의 어떤 부분에 대한 코멘트인지 즉시 확인 가능.

#### 우측 패널 — 검수 결과 6개 영역

1. **점수 + 항목별 코멘트** · 가이드 일치도 % + 콘티/카피/소구점 ✓⚠️✗ + 한 줄 코멘트 + 가이드 항목 번호 인용
2. **개선 카피 + 근거** · 기존 카피 / 개선안 / 근거 (가이드 ID + 사내 타율 상위 콘텐츠 패턴)
3. **씬 구성 (이미지·영상 배치)** · 5개 씬 타임라인 (0-3-6-10-13-15s) — 각 씬에 무엇을 보여줄지
4. **타사 콘텐츠 비교** · 유사 컨셉 광고 3건 (썸네일 + 유사도 + 운영 일수)
5. **사내 타율 상위 비교** · 우리 ROAS 상위 10% 콘텐츠와 차이점 + 예상 ROAS 시뮬레이션
6. **모듈 이동 칩** · `갤러리에서 유사 광고 보기` / `대시보드에서 시뮬레이션` / `개선안 다운로드`

**데모 동작** — 실제 LLM 연결 없음 (오늘 시연용). 사용자가 무엇을 입력하든, "검수 시작" 클릭 시 미리 작성해둔 답변 1세트가 0.5초 로딩 후 단계적으로 reveal. 추후 Claude Opus 연결 계획.

**사내 가이드 데이터** — 사용자에게 **비공개**인 백그라운드 컨텍스트. 어시스턴트 답변에 가이드 ID로 인용됨 (예: "가이드 6.1 위반"). 가이드 전문은 빌드 패키지 부록으로 제공.

### 6.4 대시보드

> `A layout · KPI + causal + user data`

**목적** — 결과 지표 ↔ 선행 지표 ↔ 유저 행동 데이터의 인과 사슬을 한 화면에서 시각화 + 자연어로 묻기.

**레퍼런스** — Amplitude Web Analytics (KPI 분류 체계, 차트 스타일) · Mixpanel Spark AI (자연어 챗봇) · Power BI Q&A · Amplitude AI Agent (답변 UX)

**레이아웃 — A · 위→아래 클래식**

1. 상단 KPI 카드 6개
2. 결과→선행 지표 인과 분해 (4개 탭)
3. 유저 데이터 분석 섹션 (Amplitude 배지)
4. 연결점 인사이트 박스
5. 하단 AI 인사이트 카드 3개

#### ① 상단 — KPI 카드 6개

매출 · ROAS · 광고비 · CTR · CVR · 신규 가입자.

각 카드에 어제/지난주 대비 변화율. **카드 클릭 시 우측에서 사이드뷰 슬라이드 인** — 시계열 차트 + 연결된 지표들 + 세그먼트 분해.

#### ② 중단 — 인과 분해 4탭

상단에 4개 탭 — `가 · Sankey` / `나 · 노드 그래프` / `다 · Funnel + 분기` / `라 · 멀티-레이어 노드`

기본 활성화 = **가 (Sankey)**. 다른 탭은 클릭 시 시각화 형태가 바뀜.

모든 시각화는 노드/요소 클릭 시 사이드뷰로 디테일 인사이트 제공.

#### ③ 유저 데이터 섹션

상단에 작은 배지 `via AMPLITUDE`로 연동 출처 명시. 3개 차트:

- 유저 행동 Funnel (방문 → 회원가입 → 첫 구매 → 재구매)
- 코호트 리텐션 (D1/D7/D30)
- 세그먼트별 전환율 (성별·연령·유입 채널)

#### ④ 연결점 인사이트 박스

유저 섹션과 결과/선행 지표를 잇는 한 줄 박스 —

> "ROAS 3.8을 만든 핵심 코호트는 30대 여성 세그먼트 — 화장솜 카테고리 콘텐츠에 가장 높은 전환."

**이 박스가 Amplitude 연동의 진짜 가치를 시각적으로 강조하는 곳.**

#### ⑤ 하단 — AI 인사이트 카드 3개

상시 노출. 클릭 시 AI 사이드패널이 그 인사이트를 컨텍스트로 자동 분석 시작.

예시 인사이트:
- "ROAS 18% 하락 — 화장솜 콘텐츠 피로도 원인"
- "신규 가입 코호트 D7 retention +12%p"
- "YouTube 광고 비용 효율 +24%, 비중 확대 권장"

**Amplitude 연동 표시** — 우상단에 `● Connected to Amplitude` 배지. 실제 연동은 없음 — 메시지로만 강조.

**빌드 도구** — KPI 카드 + 유저 섹션 차트는 **Lovable**에서. 인과 분해 4탭 시각화는 **Claude Code**로 별도 빌드 (D3.js / Recharts) 후 통합.

### 6.5 AI 사이드패널

> `cross-cutting · floating · context-aware`

**목적** — 모든 모듈을 관통하는 AI 두뇌. 현재 보고 있는 컨텍스트를 인지해 다른 답변을 주고, 답변 안에서 모듈 이동 액션까지 실행.

**트리거** — 좌측 하단 floating 버튼 (Notion AI 스타일 ✦ 아이콘). 모든 페이지에 항상 노출. 클릭 → 우측에서 사이드패널 슬라이드 인 (380~420px wide).

**컨텍스트 인지** — 현재 탭에 따라 추천 질문이 다름:

- **갤러리 탭**: "이 광고들의 공통 패턴은?"
- **어시스턴트 탭**: "이 카피, 우리 가이드와 얼마나 일치해?"
- **대시보드 탭**: "지난주 대비 ROAS 떨어진 원인?"
- **홈**: "오늘 마케팅팀이 알아야 할 것?"

**답변 형식 (4가지 종합)**

1. **인사이트 텍스트** — 한두 단락 자연어 답변
2. **인라인 트렌드 그래프** — mini line/bar chart 자동 렌더
3. **상위 콘텐츠 카드** — 썸네일 + 메트릭, 클릭 시 갤러리/어시스턴트로
4. **모듈 이동 칩** — `🎨 어시스턴트로 보내기` / `🖼 갤러리 비슷한 광고` / `📊 대시보드 시뮬레이션`

**데모 동작** — 시연용 미리 짠 답변 3~5세트. 사용자가 추천 질문 클릭 → 0.5초 로딩 → 답변 단계적 reveal (스트리밍 효과). 자유 입력 시에는 가장 가까운 답변 호출.

**비주얼** — 살짝 어두운 글래스 배경 (backdrop-blur). 답변은 말풍선 형태가 아닌 **일관된 카드 흐름**으로 — Amplitude AI Agent의 답변 UX와 유사.

## §07 시연 시나리오

90초 발표. **A 흐름** — 마케터의 일상 워크플로우를 따라가며 4개 모듈을 자연스럽게 시연. 진행하면서 B 흐름 (문제 발견 → 해결)으로 변형 가능.

### Prism 시연 — 마케터의 하루를 90초에 압축한다

#### 0–15s · HOME · OPENING
**홈 — 오늘의 AI 브리핑**

> "마케터의 아침입니다. Prism이 어제 일을 한 단락으로 정리해 줍니다. 새 경쟁사 광고 8건, ROAS는 +12%, 검수 대기 3건, 그리고 — 화장솜 콘텐츠 피로도가 보이네요."

#### 15–35s · GALLERY · RESEARCH
**갤러리 — 새 광고 + AI 패턴 분석**

> "갤러리로 갑니다. 어제 새로 들어온 광고 8개 중 3개가 비슷한 컨셉이에요. 5개 선택, AI 패턴 분석. 공통 후크 — Before/After. 이걸 우리 콘텐츠에 적용해보고 싶어요. '어시스턴트로 보내기.'"

#### 35–65s · ASSISTANT · PLAN
**어시스턴트 — 풀패키지 검수 결과**

> "기획서를 채팅바에 던지고, 검수 시작. 가이드 일치도 73%. 좌측엔 제 기획서 풀 본문, 우측엔 6개 영역 — 점수, 개선 카피와 근거, 씬 구성, 타사 비교, 사내 타율 상위 비교. 스크롤만 내려도 이만큼이 분석됩니다."

#### 65–90s · DASHBOARD · ANALYZE
**대시보드 — 인과 분해 + 유저 + 인사이트**

> "마지막. 대시보드. ROAS 3.8을 만든 코호트, 인과 분해 — 광고에서 행동, 행동에서 전환까지 한 화면에. 그리고 Amplitude 유저 데이터. 이 두 가지가 한 화면에서 만납니다. 마케터가 직접 만들고, 직접 측정하고, 직접 결정합니다 — 이게 Prism입니다."

### 리허설 메모

발표 직전 모든 모듈 1회 클릭 통과 시켜 캐시 워밍. AI 사이드패널은 시연 중 1번만 띄워서 컨텍스트 인지 보여줌 (대시보드 단계). 90초 안에 4개 탭을 다 보여주려면 **클릭은 빠르게, 설명은 짧게.**

## §08 Out of Scope

오늘 만들지 않는 것을 명확히 — 이 경계가 흐려지면 빌드 시간이 무한히 늘어난다. **"보여주기"에 집중**, 백엔드는 미래의 일.

### ✓ 오늘 만든다 (IN)

- 4개 탭 + AI 사이드패널 동작
- 갤러리 그리드 + 필터 + 카드 디테일
- 어시스턴트 풀패키지 답변 (1세트, 더미)
- 대시보드 KPI + 유저 섹션 + 인과 4탭 + AI 인사이트
- 모듈 간 컨텍스트 전달 칩 (시연용)
- 반응형 (데스크톱·태블릿)
- 비주얼 디자인 폴리싱 (Amplitude 톤+)

### ✗ 오늘 만들지 않는다 (OUT)

- 실제 백엔드 API · DB 연동
- 실제 Amplitude 데이터 통합
- 실제 LLM 연결 (Claude Opus 추후)
- 사용자 인증 / 계정 시스템 / 권한
- 데이터 영속성 (새로고침 시 리셋 OK)
- 모바일 최적화 (~767px)
- 실제 광고 자동 크롤링 / Meta API 연동
- 국제화 / 다국어
- 접근성 풀 준수 (WCAG AA 가능한 만큼만)

> **중요**: "보여지는 것"에 집중. 모든 데이터는 하드코딩된 JSON. 사용자가 클릭하든 검색하든 그럴듯한 답변이 나오면 충분. **할루시네이션 OK** — 시연용 최상의 답변만 제공.

## §09 완료 기준

Layer별 완료 기준. 각 레이어 끝나는 시점에 시연 가능한 상태. 시간이 부족해도 어느 레이어든 발표 가능.

### Layer 1 · MVP `~60분 · 누적 1시간`

- [ ] 4개 탭 (홈/갤러리/어시스턴트/대시보드) 동작 — 클릭 시 화면 전환
- [ ] 홈에 환영 메시지 + AI 브리핑 + 모듈 진입 카드 4개 표시
- [ ] 각 모듈에 placeholder 화면 표시 (빈 그리드/채팅바/KPI 카드)
- [ ] AI 사이드패널 floating 버튼 (좌하단) — 클릭 시 패널 열림
- [ ] 비주얼 톤 — Amplitude 스타일 (그라데이션, 큰 한글 타이포, 흰색 카드)
- [ ] 반응형 (데스크톱·태블릿)

→ **이 시점 시연 가능: "이런 제품입니다"**. 화면 전환·톤·구조 보여주기.

### Layer 2 · Detail Build `+60분 · 누적 ~120분`

- [ ] 갤러리: 30+개 광고 카드 + 필터 + 카드 클릭 시 디테일 패널
- [ ] 어시스턴트 진입 — 채팅바 우선 + paperclip 통합 + 시작 카드 5개
- [ ] 어시스턴트 검수 결과 — 좌측 풀본문 sticky + 우측 6개 영역
- [ ] 대시보드 — KPI 6 + Sankey 1개 + 유저 섹션 + 연결점 + 인사이트 카드 3
- [ ] AI 사이드패널 — 컨텍스트 인지 추천 질문 + 답변 1세트
- [ ] 데모 콘텐츠 채워짐 (광고 30개, 검수 결과 1세트, 대시보드 데이터)

→ **실제 발표 가능 수준**. 각 모듈이 진짜 작동, 시연 90초 흐름 가능.

### Layer 3 · Polish & Custom `옵션 · +60분 · 누적 ~180분`

- [ ] 인과 분해 4탭 시각화 (Sankey/노드/Funnel/멀티-레이어) — D3 또는 Recharts
- [ ] KPI 카드 클릭 시 우측 사이드뷰 슬라이드 인
